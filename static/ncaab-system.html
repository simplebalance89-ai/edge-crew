STEP 1 Grok Reasons Freely

Grok gets the full game context -- injuries, matchups, trends, sharp money signals, line movement -- and reasons through it with no JSON constraints. Pure analysis. 2M context window means nothing gets cut.

STEP 2 DeepSeek Formats

DeepSeek-V3.2 takes Grok's raw analysis and structures it into clean JSON: matrix_scores (all 20 variables), edge_question, letter grades. This separation means Grok thinks freely while DeepSeek handles the schema.

STEP 3 Server Recalculates

The server never trusts GPT's math. It takes the 20 matrix scores, applies the real weights, computes: (weighted_sum / max_possible) * 10. Then it runs chain detection for bonus signals. The composite score you see is always server-calculated.

FALLBACK gpt-4.1

If the two-model pipeline fails (Grok timeout, DeepSeek parse error), a single-model fallback using gpt-4.1 catches the game so nothing gets dropped.

Chain Detection Bonuses

When multiple signals align, the server detects compound patterns and adds bonus points to the composite:

TEMPO TRAP +1.0
tempo_matchup ≥ 8 AND sharp_vs_public ≥ 7 AND three_pt_reliance ≥ 7
Pace mismatch + sharp money + 3PT variance = the book undervalues tempo.
TOURNEY FADE +1.0
b2b_fatigue ≥ 7 AND bench_depth ≤ 4 AND home_court ≥ 7
Tired team, thin bench, opponent has home court. Fade the favorite.
FORM WAVE +1.5
recent_form ≥ 8 AND offensive_efficiency ≥ 7 AND ats_trend ≥ 7 AND h2h_season ≥ 6
Hot team firing on all cylinders with ATS momentum. Highest bonus -- hardest to trigger (4 conditions).

Alyssa's Math Engine

MATH ENGINE Independent from the AI pipeline. Pure expected value math:

  • Mathurin Screener -- pulls last 5 game logs from ESPN. Computes the FLOOR (lowest of L5) vs the book's line. Floor > line = mispriced.
  • Multi-Book Comparison -- lines from 12+ sportsbooks across us, us2, eu, uk, au regions. Flags gaps > 10 cents ML or > 0.5 spread.
  • Injury Gap Detection -- star OUT, identify which role players absorb minutes/usage. If the book hasn't adjusted their props = free money.
  • EV Calculation -- (our_prob * payout) - (1 - our_prob). Positive EV = bet. Negative = pass.

The Rule

The Mathurin Test: Before placing any bet -- "Can you show a NUMBER that proves the book is wrong?"

YES: "His L5 floor is 21, book has 17.5. Gap is +3.5." → BET IT.
NO: "I think they'll cover." → PASS. That's an opinion, not an edge.

No opinions. Only math. The board IS the data.
