STEP 1 Grok Reasons Freely

Grok gets the full game context -- goalie confirmations, injury reports, Corsi/xG data, line movement, sharp money signals -- and reasons through it with no JSON constraints. Pure analysis. 2M context window means nothing gets cut.

STEP 2 DeepSeek Formats

DeepSeek-V3.2 takes Grok's raw analysis and structures it into clean JSON: matrix_scores (all 21 variables), edge_question, letter grades. This separation means Grok thinks freely while DeepSeek handles the schema.

STEP 3 Server Recalculates

The server never trusts GPT's math. It takes the 21 matrix scores, applies the real weights, computes: (weighted_sum / max_possible) * 10. Then it runs chain detection for bonus signals. The composite score you see is always server-calculated.

FALLBACK gpt-4.1

If the two-model pipeline fails (Grok timeout, DeepSeek parse error), a single-model fallback using gpt-4.1 catches the game so nothing gets dropped.

Chain Detection Bonuses

When multiple signals align, the server detects compound patterns and adds bonus points to the composite:

THE MISPRICING +1.0
star_player_status ≥ 8 AND line_movement ≤ 4 AND sharp_vs_public ≥ 7
Star out + sharp money disagrees + line hasn't moved. The book is sleeping on a key absence.
GOALIE EDGE +1.5
goalie_confirmed ≥ 8 AND save_pct_trend ≥ 7 AND goalie_fatigue ≥ 7 AND corsi_xg ≥ 7
Confirmed starter, hot save %, rested, strong underlying numbers. Highest bonus -- hardest to trigger (4 conditions).
FATIGUE FADE +1.0
b2b_fatigue ≥ 8 AND road_trip_length ≥ 7 AND depth_injuries ≥ 6
B2B + long road trip + depth issues. Fade the tired team.
TRAP GAME +0.5
sharp_vs_public ≤ 3 AND line_movement ≥ 8
Public piling on one side but sharps + line movement disagree. Classic trap.
THE REGRESSION HAMMER +1.0
puck_luck ≥ 8 AND corsi_xg ≥ 7 AND sharp_vs_public ≥ 7
PDO extreme + weak xG process + public bias on wrong side. All 4 consensus models flagged this as a top-tier combo. The fade is automatic.
BACKUP FATIGUE FADE +1.0
goalie_confirmed ≤ 4 AND b2b_fatigue ≥ 8 AND sharp_vs_public ≥ 7
Backup goalie confirmed + B2B fatigue + sharp money against. Markets rarely discount enough for this combo.
THE INJURY CASCADE +0.75
depth_injuries ≥ 7 AND def_ranking ≤ 4 AND pp_pk ≥ 7
Key defensive injuries + declining high-danger against metrics + PP/PK matchup exploit. Missing top-4 D has cascading effects the market underprices.

Chinny's Props Engine

MATH ENGINE Independent from the AI pipeline. Pure expected value math:

  • Mathurin Screener -- pulls last 5 game logs from ESPN. Computes the FLOOR (lowest of L5) vs the book's line. Floor > line = mispriced.
  • Multi-Book Comparison -- lines from 12+ sportsbooks across us, us2, eu, uk, au regions. Flags gaps > 10 cents ML or > 0.5 spread.
  • Injury Gap Detection -- star OUT, identify which role players absorb minutes/usage. If the book hasn't adjusted their props = free money.
  • EV Calculation -- (our_prob * payout) - (1 - our_prob). Positive EV = bet. Negative = pass.

The Rule

The Mathurin Test: Before placing any bet -- "Can you show a NUMBER that proves the book is wrong?"

YES: "Starter's GSAx is +8.2, backup is -3.1, line only moved half a goal. Gap is massive." → BET IT.
NO: "I think they'll cover." → PASS. That's an opinion, not an edge.
